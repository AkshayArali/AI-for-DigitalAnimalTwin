# Data Sources

The raw and processed data files are **not** stored in this repo (they are too large for GitHub). To obtain the data, run the download script from the project root; it fetches from the canonical URLs below and writes into `data/raw/` and `data/processed/`.

```bash
python src/data_download.py
```

The script is the single source of truth for reproducibility: same code path and URLs produce the same derived files.

## Primary Datasets (used by `src/data_download.py`)

### 1. Tox21 (MoleculeNet mirror)

| Field | Value |
|-------|-------|
| **URL** | `https://deepchemdata.s3-us-west-1.amazonaws.com/datasets/tox21.csv.gz` |
| **Records** | ~8,000 compounds |
| **Labels** | 12 nuclear-receptor and stress-response pathway assay outcomes (binary) |
| **Format** | CSV (gzip); SMILES + 12 label columns |
| **Use** | *In vitro* HTS readouts — the core "digital twin" feature set |

### 2. NCI Acute Toxicity Database — Rat oral LD50

| Field | Value |
|-------|-------|
| **URL** | `https://cactus.nci.nih.gov/download/acute-toxicity-db/LD50_dataset.csv` |
| **Source** | NCI CACTUS Chemical Structure Lookup Service |
| **Reference** | Zhu et al., *Chem. Res. Toxicol.* 2009 |
| **Records** | ~80k compounds, 59 endpoints; script uses rat oral LD50 subset (~10k with SMILES) |
| **Format** | CSV (SMILES + endpoint columns, including `rat_oral_LD50_(?log(mol/kg))`) |
| **Use** | *In vivo* ground-truth labels — binary toxic/non-toxic from GHS 2000 mg/kg cutoff |

## Derived Files (in `data/processed/`)

Generated by `src/data_download.py`:

| File | Description |
|------|-------------|
| `tox21_clean.csv` | Tox21 with valid SMILES, de-duplicated |
| `ld50_labels.csv` | NCI rat oral LD50 subset: SMILES, MW, ld50_mgkg, binary `toxic` label |

Other pipeline steps may later produce:

| File | Description |
|------|-------------|
| `merged_features.csv` | Combined: Morgan fingerprints + assay readouts + LD50 label |
| `train.csv` / `val.csv` / `test.csv` | Scaffold-aware splits |

## Reproducibility

- **Access date:** Record the date when you run the download script (the script logs it; e.g. "Access date: YYYY-MM-DD").
- **Python environment:** See project root `README.md` for Python version, virtual environment setup, and `requirements.txt`.
- **Download script:** `python src/data_download.py` (run from project root).
